ROOT before reach.

HyphaMind is the consent-first orchestration spine for multi-model systems. Every run starts at root.digest, gathers receipts across the metabolic loop, and lands with exports your auditors can trust.

Receipts as capability

HyphaMind anchors every operation in a ROOT receipt before any network reach. VALVE decisions reference those receipts, so refusal codes are legible and reproducible.

  • ROOT digest signed
  • Capability receipts bundled
  • VALVE verdict + code

Metabolic loop discipline

The metabolic loop keeps agents honest: ROOT → VALVE → CLASP → TRACE → MIRE → SCAR → COMPOST → SPORE → ROOT. Each pass enriches provenance and raises the cost of non-consent.

We never allow CLASP or TRACE spans without a present ROOT token. “ROOT before reach” is a circuit breaker, not a slogan.

EYE signals for operators

HyphaMind’s EYE thresholds surface entropy, consent drift, and anomalous spans. REFRAIN pauses flows when ECE > 0.08 until an engineer reviews the run.

Metabolic loop, grounded in receipts

The metabolic loop weaves together technical enforcement and sociotechnical consent. Each primitive produces a signed artifact that the next stage validates, so the loop can close only when provenance is complete.

  • ROOT seeds the run with a digest, grants, and provenance anchor.
  • VALVE evaluates policy, returns clear refusal codes, and records valve_decision.
  • CLASP packages consent envelopes, keeping consent_envelope_id resolvable.
  • TRACE extends spans with trace_span_id and attaches required telemetry.
  • MIRE emits events when evidence entropy exceeds tolerance.
  • SCAR stamps compliance markers on every cadence.
  • COMPOST stores immutable audit trails.
  • SPORE exports bundles only after ROOT receipts confirm consent.
If ROOT is missing, nothing moves. Consent receipts close the loop, not dashboards.
